## 1. Что это

**Финансовый контур Brave Talents** — внутренняя система, заменяющая согласование
платежей в WhatsApp и ручную сборку управленческой отчётности:

- **Система А**: заявки на оплату → маршруты согласования → казначейский реестр
  (приоритизация при дефиците) → отметка «оплачено» бухгалтером.
- **Система Б**: автоматический учёт — поступление клиента дробится на
  НДС (3098) / себестоимость (7366) / маржу (6890), проектные балансы
  Клиент → Проект → Получатель, бюджет 6890 план-факт, депозиты/резервы.
- **Проекты/сделки**: продажники заносят сделки с блогерами из базы цен,
  проджекты ведут исполнение, статусы оплат и дебиторка видны онлайн.

Дальний план CFO: это ядро, к которому позже подключится CRM (этапы сделок) —
через API, проект/клиент — связующие сущности (DECISIONS §13).

## 2. Стек и запуск

Next.js 15.5 (App Router, TS) · Prisma 6 · PostgreSQL 16 (Docker) · Tailwind 4.

```bash
npm run db:up        # Postgres в Docker (нужен запущенный Docker Desktop)
npx prisma migrate deploy   # применить миграции (безопасно)
npm run db:seed      # справочники + демо (идемпотентен, безопасен на живой БД)
npm run dev          # http://localhost:3000
npm run test:accounting     # тесты денежного движка (25 шт.)
npm run bloggers:import -- "путь/к/файлу.xlsx"   # импорт базы блогеров
```

⚠️ Windows: перед `prisma generate` останавливать dev-сервер (держит DLL).

## 5. Ключевые доменные правила

- Деньги: **BigInt тиыны** (1 ₸ = 100 тиын), никогда float. Балансы — всегда
  агрегат журнала Transaction (остатки нигде не хранятся).
- **НДС 16/116** (ставка РК 2026): единственная точка изменения — vatFromGross
  (src/lib/estimates/service.ts) + живые расчёты форм (16/116).
- **Выплаты блогерам — по четвергам** (плановые); «Срочно» (1 раб. день) — вне
  четверга. Формат работ в заявке — утверждённая опция из сметы проекта.
- **Конфиденциальность (§10)** — в SQL, не в UI: заявитель видит свои заявки;
  проект видят владелец (продажник) + прикреплённый проджект + департамент
  владельца + руководитель блока (APPROVER — проекты, где проджект из его
  департамента) + казначейская коллегия (TREASURY_BOARD — все проекты, КРОМЕ
  спецпроектов); «видят всё» только CFO/бухгалтерия (canSeeEverything).
  Единая точка правила — projectScopeFilter (src/lib/projects/scope.ts),
  любой запрос проектов обязан использовать её. Попап «Статус проекта»
  (ProjectPeek/peekProject) отдаёт полные детали только по этому скоупу.
- Переходы статусов — **claim-guard** (`updateMany where status=...`) внутри
  `$transaction`: двойной клик/гонка не задваивают деньги и статусы.
- Смета — источник правды; новая версия **автоматически пере-разносит**
  полученные поступления (ADJUSTMENT-проводки на дельты, DECISIONS §1.1).
- Прайс блогера: опции с названиями из базы; **скидка = себес-с-налогом −
  заполненный гонорар**; ограничений на скидку нет (решение CFO).
- Себестоимость Influence-сделки = гонорары блогеров + продакшн-резервы;
  резерв считается **по каждой строке (блогер × опция)** —
  EstimateLine.reserveAmount; depositAmount версии = Σ резервов строк
  (считает сервис, руками не вводится).
- **Копилки (DECISIONS §19)**: движения депозита/резерва тегируются
  Transaction.ledgerId и НЕ входят в баланс проекта. Депозит продакшна:
  приток — депозитная доля при разнесении оплат клиента, отток — выплаты
  «Продакшн-бюджет (Influence)». Резерв ком. продакшна: приток — остаток
  котла при закрытии VP-проекта (возврат при переоткрытии). Счёт 7366 =
  котлы проектов + копилки.
- Номер проекта — сквозной по компании, присваивает система (Project.number,
  уникален в entity); клиент — только из справочника (создание — отдельным
  действием createClient, дубликаты по имени не плодим).
- Жизненный цикл заявки: Черновик → На согласовании (по ступеням) →
  Одобрена → В реестре → Оплачена; + На доработке (возврат автору),
  Отклонена, Отменена. Всё в AuditLog, тайминги видны на карточке.
- **Все маршруты замыкаются на Жадыре (главбух)** — финальная ступень
  каждого вида расхода (§21): контроль денег перед реестром, при нехватке
  средств разворачивает оплату. Зарплата — только она; дивиденды — Санжар →
  Жадыра (как и было).
